Core Logic

The Two-Stage Detection Pipeline

Detection is split into a low-cost trigger stage and a high-precision classification stage, giving strong accuracy with significantly lower continuous compute demand.

Stage 1 | OpenCV MOG2

Motion Detection

Runs on every frame at camera FPS. Builds dynamic background models, filters contours, and passes only motion-positive frames forward.

Cost: roughly 2-5ms/frame on CPU.

Stage 2 | YOLO 11 + ONNX

Object Identification

Activates only when motion is detected. Performs inference, confidence filtering, and class mapping for events like person, vehicle, or animal.

Cost: roughly 50-500ms/frame depending on hardware.

Open The Black Box

A to Z AI Component Breakdown

The executive view frames AI as value transformation; this technical view expands every operational layer from raw frames to structured events.

Data Ingestion

  • RTSP/USB input captured with FFmpeg-backed decoders
  • Frame packets include camera ID and timestamps
  • Motion queues smooth burst traffic by camera

Model Architecture

  • YOLO 11 exported to ONNX Runtime for lean inference
  • Configurable confidence thresholds and class filters
  • Pool workers parallelize detections under load

Processing Pipeline

  • Centroid-based tracking assigns persistent IDs
  • Event processor fans out to MQTT and SQLite
  • Recording service merges pre/post capture buffers

Output Generation

  • API endpoints expose health, events, clips, snapshots
  • MQTT topics power Home Assistant and automation flows
  • Structured artifacts support real-time and forensic use

Operational Impact

Efficiency Benchmarks

At 10 FPS capture, continuous YOLO can consume ~180 GPU-seconds/hour. Under two-stage gating with sparse motion events, this drops to roughly ~2 GPU-seconds/hour.

Deployment Flexibility

Hardware-Agnostic Inference Support

Raaqib supports multi-platform acceleration paths with a single architecture and configurable model profiles.

CUDA

NVIDIA GPU inference path for high-throughput installations.

MPS

Apple Silicon acceleration for efficient local deployments.

TPU

Google Coral edge acceleration for low-power environments.

Model execution is selected at runtime using device configuration values such as cpu, cuda, mps, or tpu.
