RAAQIB NVR

Hackathon Jury Presentation

Privacy-first Video Intelligence for Real Environments

RAAQIB NVR is a self-hosted network video recorder that detects relevant activity, records usable evidence, and exposes clean APIs for automation.

Speaker 1

Why This Problem Matters

Traditional NVR setups are expensive and noisy

Common pain points

  • Continuous AI inference wastes compute.
  • Cloud dependency creates privacy risk.
  • Many systems produce alerts without context.
  • Integration into existing tooling is often hard.

Our design constraints

  • Run on commodity hardware first.
  • Keep latency acceptable for monitoring.
  • Store events with evidence clips and snapshots.
  • Expose clean API and MQTT hooks.
Speaker 1

Design Choice

Two-stage pipeline: motion first, object detection on demand

Stage 1

Motion Detection (OpenCV MOG2)

Runs on every frame with low CPU cost and flags candidate motion frames.

-->

Stage 2

Object Identification (YOLO ONNX)

Only runs when motion exists, reducing unnecessary accelerator usage.

In project docs, this architecture is estimated to reduce wasted inference time dramatically for low-activity scenes.

What It Already Delivers

Operational features from the current implementation

Evidence Generation

Event clips with pre/post capture and per-event snapshots.

Event Persistence

SQLite event storage with retention policy support.

CPU default, plus CUDA, MPS, and optional Edge TPU pathways.

Tracking

Web + API

Speaker 2

Integration Surface

Designed to plug into real engineering workflows

REST API

  • GET /api/status for health and system telemetry
  • GET /api/cameras for camera state and diagnostics
  • GET /api/events for timeline and filtering
  • GET /api/recordings and /api/snapshots for evidence retrieval

Automation Hooks

  • MQTT publishing for Home Assistant and Node-RED
  • Structured event payloads for downstream workflows
  • Works with local-first deployment models
