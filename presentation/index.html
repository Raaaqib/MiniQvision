03 — The Idea

Smart gating: only run AI
when something moves

Raaqib introduces a two-stage detection pipeline — a cheap motion filter gates expensive YOLO inference. Frames with no motion are discarded in milliseconds. Only frames that matter reach the neural network.

01

Two-Stage Pipeline

MOG2 background subtraction runs first — a fast CPU check. Only motion-positive frames are forwarded to YOLO 11, slashing GPU load by 98%.

02

Distributed Multiprocessing

Every component — capture, motion, detection, tracking, recording — runs as an isolated OS process. No Python GIL. True parallelism across all CPU cores.

03

100% On-Device

No cloud. No subscriptions. All inference, storage, and processing stays on your hardware. Integrates with Home Assistant via MQTT for local automation.

04

ONNX Runtime

Model weights exported to ONNX — 50× smaller than PyTorch, faster CPU inference, zero deep learning framework overhead at runtime.

05

Smart Recording

Pre-capture circular buffer ensures you never miss the moment before detection. FFmpeg H.264/H.265 encoding with automatic retention management.

06

REST API + React UI

Full FastAPI REST interface. React dashboard for live monitoring, event browsing, and clip management. OpenAPI spec auto-generated.

Stage 1 — MOG2 Background Subtraction

The Mixture of Gaussians model maintains a statistical background model per pixel. It flags deviations as foreground — no neural network required. Running on every frame at ~2ms/frame, it acts as a lightweight gatekeeper.

2% motion threshold · contour area filter · ~2–5ms

Stage 2 — YOLO 11 Object Detection

Only frames that pass Stage 1 reach the YOLO inference engine. This cuts average YOLO calls from 600/min to just 2–10/min in typical residential scenarios — a 98% reduction.

yolo11n.onnx · 80 classes · confidence threshold configurable

One coherent system.

Capture

Reads video frames from RTSP, USB (/dev/videoX), or HTTP streams via OpenCV. Produces frame packets with timestamp and camera ID.

camera/capture.py
~1–2% CPU/cam

Motion Detector

MOG2 background subtractor + contour analysis. Discards frames below motion threshold. Feeds detection queue and pre-capture buffer simultaneously.

motion/motion.py
~2–5ms/frame

Detector Pool

Worker pool running YOLO 11 via ONNX Runtime. Each worker operates independently to prevent queue overflow during motion bursts.

50–500ms/inference

Object Tracker

Centroid distance matching assigns persistent IDs across frames. Handles object disappearance with configurable timeout windows.

Merges pre-capture circular buffer with post-detection frames. Writes H.264/H.265 MP4 clips via FFmpeg subprocess. CRF quality configurable.
